              # Документация

              # Начало работы с Defina

              В компании Defina OÜ (opens new window) было принято решение уйти от "монолита" к микросервисной структуре проекта. Учитывая то что сервис является высоконагруженное системой (highLoad) - было принято решение разделить монолит на пять разных сервисов, которые имеют свою собственную зону ответственности, но при этом общаясь через REST API, могут обмениваться данными, если одна из систем по какой причине перестанет работать, это не парализует работу сервиса, и ошибку в работе сервиса будет легче найти. К тому же так проще делегировать задачи и отвественность в области разработок.

              Пример того как будет выгладеть будущая архитектура проекта

              # Account Service

              Основной инструмент данного сервиса - Defina Account. Также может быть Defina Pay для приёма оплаты по тарифным планам. Распознание, авторизация, регистрация, аутентификация (права, роли, разрешения). Связь с другими пользователями и организациями. Контакты и связи с пользователями. Связи с платежами и транзакциями. Связь с историей действий конкретного пользователя. Связь с продуктами (пользовался, использует, обновил, не обновил, удалил).

              # File storage Service

              Основной инструмент данного сервиса - Defina Cloud. Дополнительная связь с такими инструментами как: Defina Wiki, Defina Store, Mii, ShopMe, To Defina, Arved.

              Сервис предназначен для хранения и обработки файлов (аудио, видео, документы [текстовые, табличные, графические], скриптовые файлы [код], картинки).

              Облачное хранилище для большинства приложений компании.

              # Messages Service

              Основная задача сервиса - управление электронной почтой, SMS и Push-уведомлениями. Верификация сторонних e-mail адресов, смена пароля, оповещения, подписка, массовая рассылка промо материалов, реклама, напоминания, акции, предупреждения.

              В дальнейшем планируется как конструктор электронных писем для рассылки и уведомлений. Также для обслуживания почтового клиента Zefira Mail. В случае сбоев, данные сообщения будут отправлены в архив, а на мессенджер Telegram (opens new window) придёт оповещение об ошибках.

              SMS-информирование.

              # AI Service

              Основная задача сервиса - обслуживание Искусственного Интеллекта. Сервис будет обеспечивать обработку данных от ИИ, записывать опыт обучения в специальные файлы. Очень важно, чтобы сервер выдерживал большие нагрузки и обладал высокой отказоустойчивостью. Основной продукт напрямую связанный с этим сервисом: AI Defina.

              # Shopping Service

              Основная задача сервиса - обслуживание интернет-магазинов и маркетплейсов. Чтобы снять нагрузку с основного проекта, весь трафик связанный с покупками и продажами, будет перенесён сюда, в том числе и ERP ShopMe (opens new window).

              Сервис обслуживает:

              • Defina Store (opens new window),
              • Gel Store,
              • React Page (opens new window),
              • Gimtoji,
              • Mii,
              • Defina Travel,
              • Leidke tööd,
              • Tekka.

              Единый API для всех магазинов

              WARNING

              "Жёлтый кружочек", это SQLite (opens new window) и Redis (opens new window) - они срабатывают, если один из сервисов вернул 404 или 500-ю ошибку, "запоминают" данные, и передают их, когда сервис снова заработает.
